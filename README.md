# 青龙面板自动签到脚本合集

个人在[青龙面板（QinglongPanel）](https://github.com/whyour/qinglong)上运行的自动签到与每日任务脚本合集。针对青龙环境进行了深度适配，移除了本地日志落盘，接入青龙统一通知，并对敏感信息进行了打码处理。

## 📌 脚本一览表

| 脚本文件 | 运行环境 | 定时建议 (cron) | 说明 |
| :--- | :--- | :--- | :--- |
| `bilibili_task.py` | Python3 | `15 9 * * *` | **B站每日任务**：自动观看、分享视频，每日投币（助力经验满级） |
| `discuz_auto_checkin_ql.py` | Python3 | `30 8 * * *` | **宽带技术网**：Discuz! X5.0 论坛自动登录与“领取猫粮”任务签到 |
| `ql_netease_full_sign.js` | Node.js | `0 9 * * *` | **网易云音乐完整版**：普通签到+云贝任务+黑胶乐签+VIP成长值 |
| `tianyiyun.py` | Python3 | `30 4 * * *` | **天翼云盘签到**：模拟移动端登录，每日自动签到领取容量奖励 |
| `nodeseek_sign.js` | Node.js | `30 8 * * *` | **NodeSeek 签到**：原生 Curl 强袭版，完美穿透 Cloudflare 瓦解审计 |

## 🚀 快速开始

1. **新建脚本**：在青龙面板的“脚本管理”中新建对应文件，复制本仓库对应的代码粘贴保存。
2. **环境配置**：根据下方各脚本的指引，在“环境变量管理”中添加对应的变量。
3. **添加任务**：在“定时任务”中新建任务，命令填 `task <脚本文件名>`，定时规则参考上表。
4. **调试运行**：首次配置完成后，建议手动点击一次“运行”，查看日志输出以确保配置正确。

## 🛠️ 详细配置指南

### 1. B站日常任务 (`bilibili_task.py`)
模拟登录状态自动完成每日经验任务。
*   **功能特性**：随机抽取热门视频进行观看和分享，规避风控；支持并发多账号；昵称与 UID 敏感信息打码。
*   **依赖管理**：`pip` 添加 `requests`
*   **环境变量**：
    | 变量名 | 📝 是否必填 | 示例/说明 |
    | :--- | :---: | :--- |
    | `BILI_COOKIE` | **是** | 完整的 `bilibili.com` Cookie 字符串。多账号请使用 **换行** 或 **`&`** 符号分隔。 |
    | `BILI_TOSS_COIN_COUNT` | 否 | 每日投币数量，可选 `0~5`，默认 `1`（最健康的配置，防止硬币被刷空）。 |

---

### 2. 宽带技术网任务签到 (`discuz_auto_checkin_ql.py`)
自动登录 [宽带技术网](https://www.chinadsl.net) 并申请“领取猫粮”任务。
*   **功能特性**：支持账号密码内包含 `&` 或 `#` 等特殊字符；自动探测页面内动态变更的签到链接；日志支持 ANSI 颜色渲染。
*   **依赖管理**：`pip` 添加 `requests` 和 `beautifulsoup4`
*   **环境变量**：
    | 变量名 | 📝 是否必填 | 示例/说明 |
    | :--- | :---: | :--- |
    | `CHINADSL_ACCOUNTS` | **是** | 标准 JSON 数组格式：`[{"user":"账号1","pass":"密码1"},{"user":"账号2","pass":"密码2"}]` |
    | `MANUAL_TASK_URL` | 否 | 当站点改版自动探测失败时，可手动抓包填入真实的签到请求 URL。 |

---

### 3. 网易云音乐完整签到 (`ql_netease_full_sign.js`)
基于 weapi 加密算法的网易云音乐全功能打卡助手。
*   **功能特性**：双端（PC+Android）普通签到；云贝日常任务与连签奖励领取；黑胶乐签及 VIP 成长值一键领取。
*   **依赖管理**：**纯原生 Node.js 实现**，无需安装任何外部依赖。
*   **环境变量**：
    | 变量名 | 📝 是否必填 | 示例/说明 |
    | :--- | :---: | :--- |
    | `NETEASE_MUSIC_U` | **是** | 浏览器 Cookie 中的 `MUSIC_U` 字段值。多账号请使用 **`&`** 分隔。 |

---

### 4. 天翼云盘签到 (`tianyiyun.py`)
模拟移动端安全登录流程，每日获取免费扩容。
*   **功能特性**：完整的移动端 RSA 加密登录；支持多账号运行；结果通过 WxPusher 格式化推送。
*   **依赖管理**：`pip` 添加 `requests` 和 `rsa`
*   **环境变量**：
    | 变量名 | 📝 是否必填 | 示例/说明 |
    | :--- | :---: | :--- |
    | `ty_username` | **是** | 登录手机号。多账号用 **`&`** 分隔。 |
    | `ty_password` | **是** | 登录密码。多账号用 **`&`** 分隔（需与手机号一一对应）。 |
    | `WXPUSHER_APP_TOKEN` | 否 | WxPusher 应用的 Token，用于接收表格化结果推送。 |
    | `WXPUSHER_UID` | 否 | WxPusher 用户 UID。多个 UID 请用 **`&`** 分隔。 |

---

### 5. NodeSeek 自动签到 (`nodeseek_sign.js`)
NodeSeek 社区自动化签到脚本。
*   **功能特性**：采用底层 `Curl` 进程流配合标准现代浏览器 Header 序列，穿透 Cloudflare 的指纹拦截，完美解决接口因格式审计引发的 `500` 内部错误。
*   **依赖管理**：调用系统级底层组件，**无需安装依赖**。
*   **环境变量**：
    | 变量名 | 📝 是否必填 | 示例/说明 |
    | :--- | :---: | :--- |
    | `NODESEEK_COOKIE` | **是** | 完整的 `nodeseek.com` Cookie 字符串。 |
    | `NODESEEK_UA` | 否 | 自定义浏览器 User-Agent，不填则使用默认内置 UA。 |
    | `NODESEEK_SIGN_TYPE` | 否 | 签到模式：填 `fixed` 为固定鸡腿，不填或填 `random` 为试试手气（默认）。 |

## 📢 消息通知

所有脚本均已集成青龙面板的内置推送机制。只要你在青龙面板的**系统设置 -> 通知设置**中配置好了任意一种通知渠道（如钉钉、企业微信、Telegram、Server酱、Bark 等），运行结果将会自动汇总并发送给你，无需去脚本内单独配置。
